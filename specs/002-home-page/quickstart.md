# Quickstart: Home Page Dashboard

## 1. 개요 (Overview)
본 가이드는 메인 화면 대시보드 기능을 로컬 개발 환경에서 빠르게 확인하고 검증하는 방법을 설명합니다.

---

## 2. 개발 환경 설정 (Setup)

### 2.1 사전 준비 사항
- **Node.js**: 18.x 이상
- **Prisma**: `npx prisma db push` 또는 `npx prisma migrate dev`를 통해 최신 스키마가 반영되어 있어야 함.
- **포트 설정**: `GEMINI.md`에 명시된 포트 `3004` 확인.

### 2.2 의존성 설치 및 실행
```bash
# 의존성 설치
npm install

# 데이터베이스 시딩 (선택 사항: 테스트 데이터 생성)
node prisma/seed.js

# 개발 서버 실행
npm run dev
```

---

## 3. 기능 확인 (Verification)

### 3.1 메인 화면 접속
- 브라우저를 열고 `http://localhost:3004/` 주소로 접속합니다.
- 서비스 소개 문구와 통계 데이터가 정상적으로 표시되는지 확인합니다.

### 3.2 대시보드 통계 검증
- 전체 게시글 수와 사용자 수가 데이터베이스의 실제 수치와 일치하는지 확인합니다.
- 게시판 페이지(`http://localhost:3004/posts`)에서 새로운 글을 작성한 후 다시 메인 화면으로 돌아와 통계가 실시간으로 반영되는지 확인합니다.

### 3.3 목록 필터링 테스트
- 비밀글(`isPrivate: true`)을 작성한 후 메인 화면의 최근 게시물 목록에 노출되지 않는지 검증합니다.
- 좋아요를 눌렀을 때 인기 게시물 목록에 순위가 변동되는지 확인합니다.

---

## 4. 트러블슈팅 (Troubleshooting)

- **404 에러 발생 시**: `src/app/page.tsx` 파일이 존재하며 올바르게 라우팅되고 있는지 확인합니다.
- **데이터 로딩 실패 시**: Prisma Client가 생성되었는지 (`npx prisma generate`) 및 데이터베이스 연결 상태를 확인합니다.
- **스타일 깨짐 현상**: `src/app/globals.css` 또는 전역 스타일 설정이 App Router 방식에 맞게 적용되었는지 검택합니다.
