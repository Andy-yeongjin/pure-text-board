# 📋 Project Specification: Home Page Dashboard

**지침: 모든 명세서 내용은 반드시 한글로 작성합니다.**

**Feature Branch**: `002-home-page`  
**Status**: Completed (2026-02-26)  

## 1. 프로젝트 개요
- **목적**: 서비스의 첫 인상을 결정하는 메인 화면(Landing Page)을 구축하여 사용자 경험을 개선하고 서비스 현황을 한눈에 파악할 수 있도록 함.
- **주요 특징**: 서비스 통계(전체 게시글, 사용자 수) 실시간 표시, 최근/인기 게시물 프리뷰, 명확한 Call-to-Action(CTA).
- **기술 스택**:
    - **Framework**: Next.js 14+ (App Router)
    - **Language**: TypeScript
    - **Database**: PostgreSQL (Prisma ORM)
    - **Styling**: Vanilla CSS (Minimalist)

---

## 2. 상세 요구사항 (Functional Requirements)

### 2.1 접근 제어 및 라우팅 (Access & Routing)
- **Root URL (`/`)**: 기존의 리다이렉트나 빈 페이지 대신 새로 구축된 메인 화면을 노출함.
- **상태별 대응**:
    - **비로그인 사용자**: 서비스 소개 및 로그인/회원가입 버튼(CTA) 노출.
    - **로그인 사용자**: "게시판 바로가기" 버튼 및 개인화된 환영 메시지 노출.

### 2.2 서비스 대시보드 (Service Dashboard)
- **실시간 통계**:
    - 전체 등록된 게시글 수 (Public 기준).
    - 현재 가입된 전체 사용자 수.
    - [보안]: 민감한 데이터는 포함하지 않으며 단순 카운트만 노출.
- **콘텐츠 프리뷰**:
    - **최근 게시물**: 최신순으로 정렬된 5개의 공계 게시글 제목 및 작성일.
    - **인기 게시물**: 좋아요 수가 가장 많은 5개의 공개 게시글 목록.
    - [제약]: 비밀글(`isPrivate: true`)은 프리뷰 목록에서 제외함.

### 2.3 UI/UX 구성 (Interface)
- **Hero Section**: 서비스 명칭("Pure Text Board")과 핵심 가치를 담은 간단한 문구.
- **CTA Buttons**: 
    - 게스트용: `시작하기(회원가입)`, `로그인`.
    - 회원용: `글쓰기`, `전체 목록 보기`.
- **디자인 컨셉**: "Pure Text" 아이덴티티를 유지하기 위해 화려한 이미지보다는 타이포그래피와 여백을 강조한 미니멀리즘 디자인 적용.

---

## 3. 성공 기준 (Success Criteria)

### 3.1 기능적 성공
- 사용자가 루트 경로(`/`) 접속 시 404 에러나 리다이렉트 없이 메인 화면이 정상적으로 렌더링됨.
- 데이터베이스의 실제 데이터(게시글 수, 사용자 수)가 정확하게 반영되어 표시됨.
- 최근/인기 게시물 클릭 시 해당 상세 페이지로 올바르게 이동함.

### 3.2 사용자 경험 (UX)
- 비로그인 사용자가 메인 화면에서 회원가입 페이지까지 2번 이내의 클릭으로 도달 가능함.
- 페이지 로딩 속도가 1.5초 이내로 완료되어 쾌적한 첫인상을 제공함.
- 모바일 환경에서도 통계 및 게시물 리스트가 깨짐 없이 최적화된 레이아웃으로 표시됨.

---

## 4. 가정 및 제약 사항 (Assumptions & Constraints)
- **가정**: 통계 데이터는 실시간 반영을 원칙으로 하되, 성능 최적화가 필요할 경우 5~10분 단위의 캐싱을 적용할 수 있음.
- **제약**: 외부 이미지 에셋 사용을 지양하고 CSS 기반의 스타일링만으로 디자인을 완성함.
- **데이터 보안**: 공개된 게시글의 제목만 프리뷰에 노출하며, 본문 내용은 상세 페이지 진입 후에만 확인 가능하게 함.
